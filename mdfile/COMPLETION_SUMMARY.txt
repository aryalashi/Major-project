*** ENTROPY ANALYSIS ***
Formula: H(X) = -Σ p(x) × log₂(p(x))

Examples provided:
  ✓ HTTP GET request (~4.2 bits/byte → NORMAL)
  ✓ Identical bytes (~0.0 bits/byte → MINIMUM)
  ✓ Random/encrypted (~8.0 bits/byte → MAXIMUM)

Protocol thresholds:
  ✓ HTTP (port 80): alert if > 7.5 bits/byte
  ✓ DNS (port 53): alert if > 6.5 bits/byte
  ✓ HTTPS (port 443): alert if > 7.9 bits/byte
  ✓ SSH (port 22): alert if > 7.8 bits/byte


*** THRESHOLD ADAPTATION - FOUR ALGORITHMS ***

Algorithm A: EWMA (Exponential Weighted Moving Average)
  Formula: baseline(t) = α × rate(t) + (1-α) × baseline(t-1)
  Parameters: α=0.2, headroom=3.0, window=10s
  Use case: Real-time adaptation to traffic trends
  ✓ Example: Morning (1pkt/s) → Noon (5pkt/s) → Evolution

Algorithm B: MAD (Median Absolute Deviation)
  Formula: threshold = median(x) + k × MAD
  Parameters: k=2.5 (99% confidence)
  Use case: Robust to outliers; non-normal distributions
  ✓ Example: Observations [5,4,6,5,4,5,100] with outlier ignored

Algorithm C: Z-Score Adaptive
  Formula: threshold = mean + z × std_dev
  Parameters: z=1.96 (5% FP), z=2.33 (1% FP), z=3.0 (0.3% FP)
  Use case: Standard statistical approach
  ✓ Example: Tuning FP rate from 5% to 0.3%

Algorithm D: Bayesian
  Formula: P(A|obs) using conditional probability
  Use case: Context-aware with prior probabilities
  ✓ Example: Normal time vs. Active DDoS campaign
